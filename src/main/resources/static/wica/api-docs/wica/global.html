wicaStreamCreate(serverUrl, streamConfiguration, connectionHandlers, messageHandlers, options)

Returns an object that can be used to initiate a new Wica Server Sent Event (SSE) data stream from a Wica REST Server and to subsequently handle the received events.

Note: The returned object will remain in an inactive state until it is activated by a call to its activate method.

Parameters:
Name Type Description
serverUrl string

the url of the server hosting the Wica Stream.
streamConfiguration Object

the configuration of the stream which should include the stream properties (if any), the name of the channels, and associated channel properties.

Properties
Name Type Description
props Object

the properties of the stream.
channels Array

the configuration of each channel in the stream.
channels[x].name string

the name of the channel.
channels[x].props Object

the properties of the channel.
connectionHandlers Object

callbacks for handling connection state changes.

Properties
Name Type Description
streamOpened callback

called when the stream is opened (= not yet connected).
streamConnect callback

called when the stream successfully connects.
streamClosed callback

called when the stream disconnects.
messageHandlers Object

callbacks for handling data received from the SSE stream.

Properties
Name Type Description
channelMetadataUpdated callback

called when stream metadata information is received.
connectionHandlers.channelValuesUpdated callback

called when stream value information is received.
options Object

an object providing other miscellaneous configuration options.

Properties
Name Type Description
streamReconnectIntervalInSeconds number

how often the manager should attempt to reconnect with the server if there is a communication outage.
streamTimeoutIntervalInSeconds number

how often the stream's heartbeat signal need's to be received before the channel connection will be deemed to have failed.
crossOriginCheckEnabled boolean

whether this manager should perform a CORS check.

Whether the stream is to be used for plotting purposes (in which case timestamp information will be provided for every channel value update).
